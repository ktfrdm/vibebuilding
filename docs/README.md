# Документация проекта

Добро пожаловать в документацию проекта Telegram-бота!

## Структура документации

- **[PRODUCT_DESCRIPTION.md](./PRODUCT_DESCRIPTION.md)** - Описание продукта, целевая аудитория, ключевые особенности
- **[USER_STORIES.md](./USER_STORIES.md)** - Пользовательские истории и сценарии использования
- **[FEATURES.md](./FEATURES.md)** - Детальное описание функций и требований
- **[API.md](./API.md)** - Документация API, команды и интерфейсы
- **[GITHUB_SETUP.md](./GITHUB_SETUP.md)** - Инструкция по подключению GitHub и настройке релизов

## Как использовать эту документацию

1. **Начните с PRODUCT_DESCRIPTION.md** - чтобы понять общую концепцию продукта
2. **Изучите USER_STORIES.md** - чтобы понять потребности пользователей
3. **Ознакомьтесь с FEATURES.md** - чтобы понять детали реализации функций
4. **Используйте API.md** - как справочник при разработке

## Обновление документации

Документация должна обновляться параллельно с разработкой:
- При добавлении новой функции - обновите FEATURES.md и API.md
- При изменении концепции - обновите PRODUCT_DESCRIPTION.md
- При новых пользовательских сценариях - добавьте в USER_STORIES.md
