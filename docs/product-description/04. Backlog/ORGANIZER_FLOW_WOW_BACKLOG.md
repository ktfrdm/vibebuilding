# Бэклог: улучшение флоу организатора и вау-эффект

Контекст: главная ЦА — организатор; он тянет виральность (участники → новые организаторы). Флоу организатора должен быть максимально понятным и давать ощущение «вау».  
Идеи зафиксированы для приоритизации; **реализация не начата** (анализ от ассистента).

Связанные документы: [BOT_FLOW_MVP.md](../03.%20Scenarios/BOT_FLOW_MVP.md), [BOT_VOICE_AND_TEXTS.md](../03.%20Scenarios/BOT_VOICE_AND_TEXTS.md).

## 1. Проблемные места флоу (критика) → направления улучшений

### Смешение каналов ввода

**Проблема:** После старта — reply + inline «Пропустить», дальше текст, статус — то reply-кнопки («Статус»), то inline («📋 Статус»). Новичку неочевидно, что это один сценарий.

**Идеи в бэклоге:**

- Свести к одному понятному паттерну или явно подписать: «дальше пиши текстом» / «дальше только кнопки».
- Короткий onboarding в 1–2 строки после «Давай соберёмся!»: три шага (название → варианты времени одним сообщением → подтверждение списка).

### Первый экран без «Статуса»

**Проблема:** В первом меню только «Давай соберёмся!»; статус через `/svodka` или текст «Статус». Возвращающийся организатор может не найти сводку.

**Идеи в бэклоге:**

- Вторая компактная кнопка: «У меня уже есть встреча» / «Статус» без перегруза первого экрана.

### Шаг со слотами (свободный текст + LLM)

**Проблема:** Высокий когнитивный порог; при ошибке — общее «не совсем понял» без типа проблемы (дата, прошлое, формат).

**Идеи в бэклоге:**

- Микро-примеры у поля ввода; при ошибке — **тип** проблемы («не вижу дату», «всё в прошлом») без раскрытия внутренней логики.

### После подтверждения слотов (приглашение)

**Проблема:** В личке нужно осознанно «переслать» сообщение; в группе — два потока (чат + личка организатору). Не всем очевидна механика.

**Идеи в бэклоге:**

- Один блок «что делать дальше»: участники жмут кнопку; статус смотри здесь; при необходимости одна строка-подсказка про пересылку.

### Долгий путь до «выигрыша»

**Проблема:** До ощущения ценности нужно пройти несколько шагов; при сбое LLM вау не случается.

**Идеи в бэклоге:**

- Раньше показать ценность (см. раздел «Вау») или сократить путь до первого «понял тебя» момента.

### Сводка и выбор итогового времени

**Проблема:** Кнопки «Встречаемся! … (ok/total)» перегружены; неочевидно, что можно выбрать любой слот.

**Идеи в бэклоге:**

- Упростить подписи кнопок; явная подсказка «можно выбрать любой вариант».

### После рассылки уведомлений

**Проблема:** Нет явного завершения истории и CTA на повтор (вирал и retention).

**Идеи в бэклоге:**

- CTA «Собрать ещё раз» / «Пригласи друга организовать следующую встречу через бота».

### Тон и эмодзи

**Проблема:** В `BOT_VOICE_AND_TEXTS` указан тон «без эмодзи», в продукте много эмодзи — расхождение с позиционированием.

**Идеи в бэклоге:**

- Обновить гайд под текущий UI или смягчить эмодзи под выбранный сегмент.

### `/start` у бывшего участника

**Проблема:** Без deep link `meeting_` — сброс в приветствие организатора; возможен когнитивный шум для тех, кто заходил как участник.

**Идеи в бэклоге:**

- Продумать мягкий сценарий: «продолжить как участник» / напоминание активных встреч.

## 2. Вау-эффект: текущее состояние

- Потенциальный вау — **распознавание свободного текста в список слотов** с кнопками «Да! / Изменить»; откладывается на 3-й крупный шаг и зависит от LLM.
- Приглашение в группе одним сообщением с кнопкой — сильно по UX, но для пользователя Telegram часто воспринимается как «норма», а не неожиданный эффект.

---

## 3. Идеи вау-моментов (кандидаты в бэклог)

1. **Мгновенная «магия» после ввода времени:** короткий промежуточный текст («Разбираю…» → список); лёгкий человечный ремарка при успехе — усиление «бот понял меня».
2. **Превью для друзей:** блок «Так увидят участники» до отправки в чат — контроль и гордость за приглашение.
3. **Социальное подкрепление для организатора:** уведомление не только «кто-то ответил», а микро-инсайт (например, доля ответивших, лидирующий слот) — ощущение «умной доски».
4. **Момент выбора финала:** подсветка «большинство здесь» / «оптимальный слот» (эвристика) — бот помогает **решить**, а не только показывает таблицу.
5. **Финал после рассылки:** готовая строка для чата/сторис или явное «добавь в календарь» — маленький финальный вау и повод поделиться.
