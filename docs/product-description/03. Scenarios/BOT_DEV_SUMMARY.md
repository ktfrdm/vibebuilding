# Сводка для разработки

Техническая выжимка из [BOT_FLOW_MVP](BOT_FLOW_MVP.md) — сущности, действия бота, LLM, алгоритмы.

---

## Сущности

| Сущность | Поля |
|----------|------|
| **Встреча** | id, название, период, слоты (массив: дата+время), статус (создана / время назначено), чат_id (где создана), создатель_id |
| **Слот** | дата, время; у участника: отмечен как удобный или не отмечен (считается занят) |
| **Участник** | user_id, встреча_id, статус (отметил слоты / отказался «не смогу» / не ответил), ответы — только удобные слоты (слот_id в списке выбранных) |

---

## Ключевые действия бота

1. Принять название, период, слоты от организатора — **текстом**; LLM парсит период и слоты.
2. При неудачном парсинге — уточнить у пользователя.
3. Создать встречу, сгенерировать ссылку `t.me/bot?start=meeting_{id}`.
4. В групповом чате — запостить сообщение с кнопкой.
5. По ссылке — сразу показать слоты и режим ответа: участник отмечает только удобные слоты (неудобные не трогает) или нажимает «не смогу».
6. Сохранить ответы (слоты или отказ).
7. Упорядочить слоты по количеству отметивших (сначала слоты, где все отметили как удобный).
8. После выбора организатором — спросить место. Свободным отправить полное уведомление; занятым в выбранном слоте — «Сможешь прийти?»; после ответа — подтвердить или вывести из списка.

---

## Интеграция LLM

**Назначение:** Распознавать и обрабатывать ответы организатора, введённые текстом.

**Что парсит LLM:**
- **Период (шаг 3):** «в эту субботу» → дата; «25 февраля» → дата; «следующие выходные» → диапазон дат; «7–8 марта» → диапазон.
- **Слоты (шаг 4):** «суббота 12:00, 15:00, 18:00» → список [date, time]; «с 12 до 18» → интервал; «вечером», «днём» → временные окна с учётом контекста периода.

**Формат ответа LLM:** Структурированные данные (JSON): `{ period: { start, end }, slots: [{ date, time }, ...] }`. При невозможности распознать — явный ответ «не удалось», чтобы бот запросил уточнение.

---

## Алгоритм показа слотов организатору

Слоты упорядочиваются по удобству для группы:
1. **Слоты, где все ответившие отметили как удобный** — в начале списка.
2. **Остальные слоты** — по убыванию числа отметивших.

Для каждого слота показывать: сколько ✓ отметили как удобный, сколько ✗ не отметили. Организатор может выбрать любой слот. Участникам, которые не отметили выбранный слот, уходит вопрос: «[Название] назначена на [дата, время]. Сможешь прийти?»
