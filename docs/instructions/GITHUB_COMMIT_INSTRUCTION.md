# Инструкция по коммитам на GitHub

## Workflow

Локальные изменения не попадают на GitHub автоматически. Нужно выполнить три команды:

```
Изменили файлы локально
         ↓
    git add .
         ↓
    git commit -m "описание изменений"
         ↓
    git push
```

---

## Что делает каждая команда

| Команда | Что делает |
|---------|------------|
| `git add .` | Добавляет все изменённые файлы в «staging area» (подготовку к коммиту) |
| `git commit -m "текст"` | Создаёт коммит с описанием того, что вы изменили |
| `git push` | Отправляет ваши коммиты на GitHub |

---

## Пример

1. Редактируете `bot/handlers/start.py` в Cursor
2. Сохраняете файл
3. В терминале:
```bash
cd "/Users/ekaterinabazhenova/Вайб"
git add .
git commit -m "Обновил приветствие в команде /start"
git push
```

---

## Кратко

- Меняете файлы локально → сохраняете
- Чтобы отправить изменения на GitHub → добавляете, коммитите, пушите

---

## Полезная команда

Перед коммитом можно посмотреть статус:

```bash
git status
```

Она покажет, какие файлы изменены и готовы к коммиту.
